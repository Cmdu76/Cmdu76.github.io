Game Engine Entry

This article will explain how I manage the launching of the game engine and so the game itself.

The article will be split in few parts :

  • main()
  • The window and the events
  • The game loop and the update
  • States
  • An example of integration in the engine : ImGui

I – main()

Our main() function will be very simple, but I think it requires some explanations to understand properly all the content of the article.

So, here you have my main() function yet (It might have change a bit)

int main()
{
    Application app;
    app.start<MyState>();
    return 0;
}

As you can see, Application is one of the most important classes of our game engine. This class will handle a lot of things like the window, the game loop and a lot more if you want.

I could add parameters to the constructor of the application like the size of the window, but for the example I want to keep things as simple as possible.

The start() function take a template parameter. This parameter is used to set the beginning state of the game. I will explain it in detail in the States part of this article.


II – The window and the events

The previous was a kind of introduction, now let’s get in detail.

The Application class will handle the window. We have to create it with the good settings, we want to check every frame if it’s still open and more important, we will get events from it.

There are two way to get events from the OS. One is blocking and the other is not for example, in the SFML, waitEvent() is blocking and pollEvent() isn’t. We want our game engine to run at full speed and not only when an event is received, so we will use the non-blocking option.

Let’s create the events() function :

sf::Event event;
while (mWindow.pollEvent(event))
{
    mStates.handleEvent(event); // I will explain this latter
}
